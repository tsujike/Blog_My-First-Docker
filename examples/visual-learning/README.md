# 視覚的に学ぶDocker - 実践サンプル

このフォルダには、**Dockerを視覚的に理解するための実践サンプル**が含まれています。

実際に手を動かしながら、Dockerの動作を目で確認できるように設計されています。

## サンプル一覧

### 1. hello-visual/ - 最も簡単な例
**目的：** Dockerの基本動作を視覚的に確認する

**特徴：**
- シンプルなHTMLページ
- コンテナの起動・停止を視覚的に確認
- ファイルの場所を明確に表示

**学習ポイント：**
- コンテナが実際に動いていることを確認
- ファイルがどこにあるかを理解
- コンテナを削除してもファイルが残ることを確認

### 2. counter-app/ - データを永続化する例
**目的：** ボリュームの動作を視覚的に確認する

**特徴：**
- カウンターアプリケーション
- データベースを使った永続化
- コンテナを削除してもデータが残ることを確認

**学習ポイント：**
- ボリュームの必要性を理解
- データの永続化を視覚的に確認
- コンテナとデータの関係を理解

### 3. file-explorer/ - ファイルシステムを理解する
**目的：** コンテナ内のファイルシステムを視覚的に確認する

**特徴：**
- ファイル一覧を表示するWebアプリ
- コンテナ内のファイル構造を確認
- ボリュームマウントの動作を確認

**学習ポイント：**
- コンテナ内のファイル構造を理解
- ボリュームマウントの動作を確認
- ホストとコンテナのファイルの関係を理解

## 使い方

各サンプルのフォルダ内に、詳細な手順が記載されています。

**基本的な流れ：**
1. サンプルフォルダに移動
2. `docker-compose up -d` で起動
3. ブラウザでアクセスして動作を確認
4. ファイルを編集して変更を確認
5. コンテナを停止・削除して動作を確認

## 学習のポイント

### 視覚的に確認すべきこと

1. **コンテナが動いている**
   - `docker ps` でコンテナの状態を確認
   - ブラウザでアプリにアクセス

2. **ファイルの場所**
   - ホストのファイルがどこにあるか
   - コンテナ内のファイルがどこにあるか

3. **データの永続化**
   - コンテナを削除してもデータが残る
   - ボリュームの存在を確認

4. **変更の反映**
   - ファイルを編集して変更が反映されるか
   - コンテナを再起動して動作を確認
